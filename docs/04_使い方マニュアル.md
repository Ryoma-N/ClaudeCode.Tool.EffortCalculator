# 使い方マニュアル
**工数計算ツール**
最終更新：2026-03-17

---

## はじめに

このツールはIT・SES業界の案件管理に特化した工数管理ツールです。
WBS（作業分解構造）を作成し、タスクごとに日々の作業時間を記録することで、以下が自動的に計算されます。

- **進捗率**：各タスクの達成度（手動入力）を元にした加重平均
- **出来高**：見積工数 × 進捗率（本来使うべき時間）
- **乖離**：実績工数 − 出来高（＋なら予算超過傾向）
- **月次集計**：担当者別・工程別の工数サマリー

> ブラウザのIndexedDBにデータを保存します。ブラウザのキャッシュ削除でデータが消えるため、重要なデータはPDF出力で保管してください。

## 初期設定（プロジェクト開始時）
http://localhost:5173/
### Step 1: プロジェクトを作成する

1. 左メニューの **「プロジェクト」** をクリック
2. 右上の **「＋ 新規作成」** をクリック
3. 以下を入力して **「保存」**

| 項目 | 入力内容 | 例 |
|---|---|---|
| プロジェクト名 | 案件名 | ECサイトリニューアル |
| 説明 | 案件概要（任意） | 認証・商品管理・決済を刷新 |
| 開始日 | 作業開始日 | 2026/01/06 |
| 終了日（期限） | 納期 | 2026/03/31 |
| 1人日あたりの作業時間 | 通常8h（会社規定に合わせる） | 8 |
| 1人月あたりの営業日数 | 通常20日（月により調整可） | 20 |
| ステータス | 進行中 | 進行中 |

> **ポイント**：1人日・1人月の設定値が人日/人月換算表示に使われます。SES契約の単位に合わせて設定してください。

---

### Step 2: 工程を定義する

プロジェクト詳細画面右上の **「⚙️（設定）」** をクリック → **「工程定義」** セクションで登録。

**SES案件でよく使う工程例：**

| 工程名 | 対象作業 |
|---|---|
| 要件定義 | ヒアリング・要件整理 |
| 基本設計 | 機能仕様・DB設計 |
| 詳細設計 | 詳細仕様・テスト設計 |
| 製造（実装） | コーディング |
| 単体テスト | UT |
| 結合テスト | IT |
| レビュー | コードレビュー・ドキュメントレビュー |
| 管理 | PM業務・会議・報告 |

1. 入力欄に工程名を入力
2. **「＋ 追加」** または **Enter** キーで登録
3. 繰り返して必要な工程を全て登録

> **ポイント**：工程は後から追加・変更・削除できます。工程名はフィールド上で直接クリックして編集できます。

---

### Step 3: フェーズ（大分類）を作成する（任意）

フェーズはWBSの大分類です。「設計フェーズ」「開発フェーズ」のように工程ごとに分けるか、「画面単位」や「機能単位」で分けるかはプロジェクトに合わせて選択してください。

1. プロジェクト詳細画面の下部 **「＋ フェーズ追加」** をクリック
2. フェーズ名を入力して **「追加」**

> **ポイント**：フェーズを使わずに全タスクをフラットに管理することも可能です。小規模案件はフェーズなしで十分です。

---

### Step 4: タスク（WBS行）を作成する

1. **「＋ タスク追加」** をクリック
2. 以下を入力して **「保存」**

| 項目 | 入力内容 | 説明 |
|---|---|---|
| タスク名 | 作業項目名 | 例：ログイン画面実装 |
| 説明 | 詳細（任意） | |
| フェーズ | 所属フェーズ（任意） | |
| ステータス | 未着手 / 進行中 / 完了 | |
| 期日 | タスクの期限（任意） | |
| 見積工数(h) | 見積時間 | 例：16（= 2人日） |
| 進捗率 | 現時点の達成度（0-100%） | 新規なら0% |
| 担当者 | 担当者名（任意） | 例：山田 太郎 |

> **ポイント**：見積工数を入力しないとEVM計算（出来高・乖離）が機能しません。必ず入力することを推奨します。

## 日常の使い方（毎日の作業記録）

### 作業実績を記録する

作業が終わったら、その日のうちに記録します。

1. プロジェクト詳細から該当タスクをクリック → **タスク詳細画面**へ
2. **「工程別実績」** セクションの **「新規実績入力」** フォームに入力

| 項目 | 入力内容 |
|---|---|
| 日付 | 作業した日付（デフォルト：本日） |
| 工程 | 今日行った作業の工程（設計・実装・テスト等） |
| 時間(h) | 作業時間（0.5h単位で入力可） |
| 備考 | 作業内容のメモ（任意） |

3. **「＋ 記録を追加」** をクリック

> **ポイント**：1つのタスクに複数の工程で作業した場合は、工程ごとに記録を追加します。

**例：ログイン機能実装タスクで今日やったこと**
```
設計    2h  詳細設計書作成
実装    4h  APIエンドポイント実装
レビュー 1h  コードレビュー対応
```

---

### 進捗率を更新する

タスクの達成度が変わったらタスクを編集して進捗率を更新します。

1. タスク詳細画面の **「✏️（編集）」** をクリック
2. **「進捗率」スライダー** を現在の達成度に合わせる
3. **「保存」**

> **進捗率の目安**
> - 0%：着手前
> - 30%：着手済み、まだ序盤
> - 50%：大体半分終わった
> - 70-80%：ほぼ完成、確認作業中
> - 100%：完了（ステータスも自動で「完了」になります）

> **注意**：進捗率は「作業時間の消化率」ではなく「仕事の完了度」です。時間を使っても進んでいなければ低く設定してください。

---

### アラートを確認する

ダッシュボードに以下のアラートが表示されます。

| アラート | 内容 |
|---|---|
| 🔴 期限超過 | 期日を過ぎているが未完了のタスク |
| 🟡 3日以内に期限 | 期日まで3日以内の未完了タスク |

アラートの行をクリックすると該当タスクの詳細へ移動できます。

## 進捗報告の準備

### プロジェクト進捗を確認する

プロジェクト詳細画面の **進捗サマリー** で以下を確認できます。

| 指標 | 意味 | 判断 |
|---|---|---|
| 進捗率 | WBSベースの完了度（%） | これが主要な進捗指標 |
| 工数消化率 | 見積に対して実績が何%か | 80%超えたら注意 |
| 乖離 | 実績 − 出来高（h） | ＋ならコスト超過傾向 |

**乖離の読み方**

```
例）見積16h・進捗50%・実績10h の場合

  出来高 = 16h × 50% = 8h
  乖離   = 10h - 8h  = +2h

  → 50%完了するのに本来8h使うはずが、10h使ってしまっている
  → このペースだと完成時に+4h（合計20h）かかる予測
```

---

### フェーズ別グラフで分析する

プロジェクト詳細のグラフセクションに以下が表示されます。

| グラフ | 内容 |
|---|---|
| フェーズ別棒グラフ | フェーズごとの「見積」「出来高」「実績」を3本バーで比較 |
| 工程別円グラフ | 設計/製造/テスト等の作業時間の内訳 |

**棒グラフの読み方**
- **見積（グレー）**：予定の工数
- **出来高（紫）**：進捗率から計算した「相当する工数」
- **実績（緑）**：実際に費やした工数
- 実績 > 出来高 → 工数がかかりすぎ
- 実績 < 出来高 → 効率良く進んでいる

## 月次報告書の作成

### 月次レポート画面を開く

左メニューの **「月次レポート」** をクリック。

### 絞り込み設定

| 項目 | 選択内容 |
|---|---|
| 対象月 | 報告したい年月を選択（例：2026-03） |
| プロジェクト | 特定プロジェクトのみ or 全プロジェクト |

### 確認できる情報

**サマリーカード**
- 合計工数(h)・人月換算・稼働日数・記録件数

**担当者別集計**
- 誰が何時間作業したか（人月換算付き）

**工程別集計**
- 設計/製造/テスト等、どの工程に何時間使ったか（人月換算付き）

**作業明細**
- 日付・プロジェクト・タスク・工程・担当者・時間・備考の一覧

### PDF出力する

1. 絞り込み条件を設定
2. 右上の **「▼ PDF出力」** ボタンをクリック
3. `月次工数_YYYY-MM.pdf` がダウンロードされる

## プロジェクト完了処理

1. 全タスクの進捗率が100%になったことを確認
2. プロジェクト詳細の **「✏️（編集）」** をクリック
3. ステータスを **「完了」** に変更して保存
4. 必要に応じてPDFレポートを出力して保管

> 完了プロジェクトはプロジェクト一覧の「完了」タブで確認できます。

## よくある操作

### 誤って入力した実績を削除したい

タスク詳細画面 → 実績履歴の行右端の **「🗑️」** をクリック → 確認ダイアログで「OK」。

### 工程名を変更したい

プロジェクト設定 → 工程一覧の工程名部分をクリックして直接入力 → フォーカスを外すと自動保存。

### タスクを別のフェーズに移動したい

タスク詳細の **「✏️（編集）」** → 「フェーズ」セレクトで変更 → 「保存」。

### プロジェクトを削除したい

プロジェクト詳細の **「🗑️」** をクリック → 確認ダイアログで確認。
**フェーズ・タスク・実績データが全て削除されます。元に戻せません。**

### スマートフォンで使いたい

ブラウザでURLを開き、ホーム画面に追加するとPWA（アプリ）として使用できます。
スマートフォンでは主に日次実績の記録に使うことを推奨します。

## 運用のコツ

### 毎日の習慣にする

| タイミング | 操作 |
|---|---|
| 退勤前（推奨） | その日の作業実績をタスクごとに記録 |
| 週次（月曜等） | 各タスクの進捗率を更新 |
| 月末 | 月次レポートを出力して保管 |

### 見積精度を上げるには

- 作業完了後に「見積 vs 実績」の差分を必ず確認する
- 乖離がプラスになったタスクは「なぜ時間がかかったか」をメモに残す（備考欄を活用）
- 次の案件の見積時に過去の実績を参照する

### 進捗率の正直な入力を徹底する

工数消化率（実績/見積）と進捗率が大きく乖離している場合、リスクのサインです。

```
悪い例：実績60%消化・進捗率60%入力（惰性で一致させている）
良い例：実績60%消化・進捗率40%入力（正直に遅れを反映）
       → 乖離がプラス大きく表示される → 早期に対策できる
```

## 用語集

| 用語 | 説明 |
|---|---|
| WBS | Work Breakdown Structure。プロジェクトの作業を階層的に分解した構造 |
| 人日（MD） | Man Day。1人が1日作業した工数。デフォルト8時間 |
| 人月（MM） | Man Month。1人が1ヶ月作業した工数。デフォルト160時間（20日×8h） |
| 見積工数 | プロジェクト開始前に合意した作業量の目標値 |
| 実績工数 | 実際に費やした時間の記録値 |
| 進捗率 | タスクの完了度（0-100%）。作業者が手動で入力 |
| 出来高（EV） | Earned Value。見積工数 × 進捗率。「相当する価値を生み出した工数」 |
| 乖離 | 実績工数 − 出来高。プラスは効率悪化・予算超過傾向を示す |
| EVM | Earned Value Management（アーンドバリューマネジメント）。工数・進捗・コストを統合的に管理する手法 |
| フェーズ | 設計・開発・テスト等、プロジェクトの大きな区切り |
| 工程 | 設計・製造・レビュー等、作業の種類の分類 |
