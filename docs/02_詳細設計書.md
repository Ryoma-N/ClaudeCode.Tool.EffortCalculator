## 1. データモデル設計

## 2. 計算ロジック詳細

### 2.1 基本単位換算

```
1人日 = Project.mdHours 時間（デフォルト 8h）
1人月 = Project.mmDays 営業日（デフォルト 20日）
1人月 = Project.mmDays × Project.mdHours 時間（デフォルト 160h）
```

### 2.2 タスクレベルの計算（EVM）

```
// 出来高工数（Earned Value）
earnedHours = estimatedHours × progressRate / 100

// 乖離（実績 − 出来高）
variance = actualHours - earnedHours
  → + : 出来高より実績が多い（予算超過傾向・効率悪化）
  → - : 出来高より実績が少ない（予算に余裕・効率良好）
  → 0 : 予定通り

// 実績工数
actualHours = Σ WorkEntry.hours（そのタスクの全実績）
```

### 2.3 フェーズレベルの計算

```
estimatedHours = Σ task.estimatedHours（フェーズ内タスク合計）
actualHours    = Σ task.actualHours
earnedHours    = Σ task.earnedHours
variance       = actualHours - earnedHours

// 進捗率（加重平均。見積がある場合）
progress = earnedHours / estimatedHours × 100  （estimatedHours > 0 の場合）

// 見積なしタスクのみの場合：単純平均
progress = Σ task.progressRate / タスク数
```

### 2.4 プロジェクトレベルの計算

フェーズレベルと同じ計算式を全タスクに適用。

```
totalEstimatedHours = Σ task.estimatedHours（プロジェクト全タスク）
totalActualHours    = Σ task.actualHours
totalEarnedHours    = Σ task.earnedHours
totalVariance       = totalActualHours - totalEarnedHours
progress            = totalEarnedHours / totalEstimatedHours × 100
```

### 2.5 進捗バーの色分けルール

| 条件 | 色 | 意味 |
|---|---|---|
| progress >= 100% | 緑（bg-ok） | 完了 |
| progress >= 80% | 黄（bg-warn） | 完了間近 |
| progress < 80% | 紫（bg-brand） | 通常 |
| 消化率 >= 100% | 赤（bg-danger） | 予算超過 |
| 消化率 >= 80% | 黄（bg-warn） | 予算消化注意 |

### 2.6 ステータスと進捗率の連動ルール

| 操作 | 連動 |
|---|---|
| ステータス → 完了 | progressRate = 100 に自動設定 |
| progressRate = 100 | ステータス → 完了 に自動設定 |
| progressRate > 0 かつ ステータス = 未着手 | ステータス → 進行中 に自動設定 |
| progressRate = 0 かつ ステータス = 進行中 | ステータス → 未着手 に自動設定 |

### 3.2 状態管理（Zustandストア）

#### ストア構造

```typescript
{
  // データ
  projects:           Project[]
  phases:             Phase[]
  tasks:              Task[]
  processDefinitions: ProcessDefinition[]
  workEntries:        WorkEntry[]
  isLoaded:           boolean

  // アクション（CRUD）
  loadAll()             // 初期読み込み
  addProject()          // プロジェクト追加
  updateProject()       // 更新
  deleteProject()       // 削除（フェーズ・タスク・実績をカスケード削除）
  addPhase() / updatePhase() / deletePhase()
  addTask() / updateTask() / deleteTask()
  addProcess() / updateProcess() / deleteProcess()
  addWorkEntry()        // 日次実績1件追加
  updateWorkEntry()     // 更新
  deleteWorkEntry()     // 削除

  // 計算セレクター
  getTaskWithActuals(taskId)    // タスク + 実績計算結果
  getProjectWithStats(projectId) // プロジェクト + EVM計算結果
  getAllProjectsWithStats()      // 全プロジェクト
  getAlertTasks()               // 期限超過・間近タスク
}
```

#### 無限ループ防止

Zustandのセレクター内で計算関数を直接呼ぶと参照が変わるたびに再レンダリングが発生する。
**必ず `useMemo` でラップ**して使用する。

```typescript
// NG（無限ループ）
const stats = useAppStore(s => s.getProjectWithStats(id))

// OK
const getStats = useAppStore(s => s.getProjectWithStats)
const stats = useMemo(() => getStats(id), [getStats, id, projects, tasks, workEntries])
```

## 4. 削除時のカスケード処理

| 削除対象 | 連動削除 |
|---|---|
| Project | Phase・Task・ProcessDefinition・WorkEntry（全て） |
| Phase | タスクのphaseIdをクリア（タスク本体は削除しない） |
| Task | WorkEntry（そのタスクの全実績） |
| ProcessDefinition | WorkEntryは削除しない（孤立エントリとして残る） |
| WorkEntry | 単体削除のみ |
