# 基本設計書
**システム名：工数計算ツール**
最終更新：2026-03-17

---

## 1. システム概要

### 1.1 目的

IT・SES業界における案件の工数見積もり・進捗管理・実績記録を一元管理するWebアプリケーション。
WBSベースのタスク管理と、アーンドバリュー（EVM）方式の進捗指標を組み合わせ、見積乖離の早期検知と月次報告書の作成を支援する。

### 1.2 対象ユーザー

| ユーザー | 用途 |
|---|---|
| プロジェクトマネージャー / TL | 見積作成・進捗確認・月次報告書作成 |
| 開発メンバー | 日次作業実績の記録・タスク進捗更新 |
| 管理職・営業 | プロジェクト横断の工数確認（ダッシュボード） |

> **現フェーズ**：シングルユーザー前提。将来的にSupabase移行によるチーム共有を想定した設計。

### 1.3 スコープ

**対象**
- プロジェクト管理（複数案件・ステータス管理）
- WBS管理（フェーズ・タスク階層）
- 工程定義管理（設計/開発/テスト等）
- 日次作業実績記録
- EVM方式進捗計算（進捗率・出来高・乖離）
- 月次集計・レポート
- PDF出力
- PWA（スマートフォン対応）

**対象外**
- マルチユーザー認証（将来対応）
- ガントチャート（将来対応）
- コスト・単価管理（将来対応）
- 外部システム連携（将来対応）

## 3. 機能一覧

### 3.1 機能概要

| 機能ID | 機能名 | 概要 |
|---|---|---|
| F01 | プロジェクト管理 | 案件の作成・編集・削除・ステータス管理 |
| F02 | フェーズ管理 | WBS大分類（設計フェーズ・開発フェーズ等）の管理 |
| F03 | タスク管理 | WBS詳細行の作成・編集・担当者・進捗率管理 |
| F04 | 工程定義管理 | 作業工程（設計/製造/テスト等）の定義 |
| F05 | 日次作業実績 | 日付・工程・時間・備考の日次記録 |
| F06 | 進捗計算（EVM） | 出来高・乖離・進捗率の自動計算 |
| F07 | ダッシュボード | 全プロジェクトの概況・アラート表示 |
| F08 | 月次レポート | 月次工数集計・担当者別/工程別集計・PDF出力 |
| F09 | プロジェクトPDF | プロジェクト単位の工数報告書PDF出力 |
| F10 | PWA | スマートフォンからの日次入力対応 |

### 3.2 アラート機能

| アラート種別 | 条件 | 表示色 |
|---|---|---|
| 期限超過 | タスクの期日 < 本日、かつステータスが完了以外 | 赤 |
| 期限間近 | タスクの期日まで3日以内、かつ完了以外 | 黄 |

## 4. 非機能要件

### 4.1 パフォーマンス

- 初期表示：2秒以内（ローカルDB読み込み含む）
- 操作応答：即時（状態管理はインメモリ）

### 4.2 データ保全

- ストレージ：ブラウザ IndexedDB（ローカル）
- バックアップ：ブラウザ依存（将来：エクスポート機能）
- データ消失リスク：ブラウザデータ削除時に消失

### 4.3 対応環境

| 環境 | 対応状況 |
|---|---|
| Chrome / Edge（最新版） | ◎ 推奨 |
| Firefox（最新版） | ○ 対応 |
| Safari（最新版） | ○ 対応 |
| スマートフォン（Chrome/Safari） | ○ PWAとして動作 |

### 4.4 セキュリティ

- 認証：なし（シングルユーザー想定）
- データ：全てブラウザローカルに保存。外部送信なし
